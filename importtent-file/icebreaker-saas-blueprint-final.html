Architecture

How the Layers Connect

Layer 1 — Routes

src/app/

Pages & API route handlers. Thin layer — receives requests, calls features, returns responses. No business logic here.

Layer 2 — Features

src/features/

Domain logic grouped by feature: AI generation, web scraping, billing, queue processing, SEO. Each module is self-contained.

Layer 3 — Components

src/components/

Shared React UI organized by purpose: primitives, layout, forms, results display, billing widgets. Used by route pages.

Layer 4 — Infrastructure

src/lib/

Database clients, auth helpers, Redis, validation, rate limiting. Shared across all features. No business logic — only infrastructure.

Layer 5 — Hooks & Types

src/hooks/ & src/types/

Custom React hooks for client-side logic (auth, credits, job polling). TypeScript types split by domain for clean imports.

Layer 6 — Data & Config

prisma/ & scripts/

Database schema & migrations live outside src/. Scripts for seeding and SEO content generation run at build/dev time.

IMPORT DIRECTION RULE

Routes (app/) → Features → Lib (infra)
Routes (app/) → Components → Hooks / Types

Never import from app/ into features/. Never import from features/ into lib/. Dependencies flow downward only.

FEATURES/

Domain logic grouped by feature. Each folder is self-contained — AI, scraper, billing, queue, SEO.

LIB/

Infrastructure code shared across features — database clients, auth, Redis, validation, rate limiting.

COMPONENTS/

UI layer organized by purpose — primitives, layout, forms, results, billing widgets.

// .env.local — in src/lib/db/prisma.ts singleton

✘ WRONG (direct — will exhaust connection limits):

✔ CORRECT (pooled via PgBouncer):
DATABASE_URL="postgresql://user:pass@host:6543/postgres?pgbouncer=true"

// For migrations only:
DIRECT_URL="postgresql://user:pass@host:5432/postgres"

// prisma/schema.prisma
datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")      // Pooled for queries
  directUrl = env("DIRECT_URL")       // Direct for migrations
}

Bottom line: Always port 6543 for app, port 5432 for migrations only. This prevents the #1 cause of Supabase crashes in serverless.

Critical Production Gotchas

5 Risks That Will Break Your SaaS If Ignored

⏱️

Risk #1: Vercel Timeout Trap

Vercel serverless functions timeout at 10-60 seconds. Processing 500 GPT-4o-mini calls synchronously will always fail. Fix: Upstash Redis queue in Phase 3 (already integrated into the build plan above).

💸

Risk #2: AI API Costs vs Fixed Plans

GPT-4o-mini costs ~$0.00016/generation. On Pro plan: 500 × $0.00016 = $0.08 (93% profit margin on $29). But you still need hard caps. Fix: Hard credit limits + UsageLog tracking in src/features/billing/usage-tracker.ts.

NEVER offer unlimited AI generation on a fixed monthly fee.

🚫

Risk #3: Rate Limiting on Website Scrapers

100 leads from the same domain = 100 rapid scrapes = IP blocked. Fix: 24hr ScraperCache with domain-level dedup in src/features/scraper/cache.service.ts. Scrape once per domain per day. Cleanup cron at src/app/api/cron/cleanup-cache/.

📊

Risk #4: No Visibility Into AI Costs Per User

Without per-user cost tracking, you can't identify abusers or optimize pricing. Fix: UsageLog model tracking tokensUsed, estimatedCost, and model per generation. Build a cost dashboard with groupBy userId queries.

⏳

Risk #5: Lemon Squeezy 13-Day Hold Period

First payments are held for 13 days (fraud prevention). Day 1 → Day 14 release → Day 15-18 to bKash/BRAC. Plan for it: Budget for 2-3 weeks before first payout. After that, payouts on 1st & 15th monthly.

Update #1: LinkedIn URL Edge Case CRUCIAL

Leads often provide LinkedIn URLs instead of company websites. Scraping LinkedIn = instant IP ban.

✓ FIX — src/features/scraper/linkedin-detector.ts

function isLinkedInUrl(url: string): boolean {
  const patterns = ['linkedin.com/in/', 'linkedin.com/company/'];
  return patterns.some(p => url.toLowerCase().includes(p));
}

// In scraper.service.ts — skip scrape, fall through to title+company prompt
if (isLinkedInUrl(lead.website)) return null;

Result: Users can upload CSVs with LinkedIn URLs without breaking your system.

📧

Update #2: Deliverability & Spam Protection VALUE ADD

If AI generates spam-trigger words ("Guarantee," "Free," "Buy now"), emails land in junk. This kills trust.

✓ FIX — src/features/ai/spam-checker.ts

const SPAM_TRIGGERS = [
  'guarantee', 'free', 'buy now', 'act now',
  'limited time', 'risk-free', 'click here', ...
];

export function checkSpamTriggers(text: string) {
  const flagged = SPAM_TRIGGERS.filter(w => text.toLowerCase().includes(w));
  return { isClean: flagged.length === 0, flaggedWords: flagged };
}

// Show ⚠️ SPAM RISK badge in src/components/results/SpamBadge.tsx

Why it matters: Positions your product as "deliverability-aware" — a free differentiator vs generic AI wrappers.

🔍

Product Hunt = 1-3 day spike. Twitter = constant hustle. You need an evergreen channel.

Prisma + serverless = new DB connection every invocation. Under load → "too many connections" crash. Already integrated into Phase 1 — use pooled port 6543, direct port 5432 only for migrations. The Prisma singleton lives in src/lib/db/prisma.ts.
